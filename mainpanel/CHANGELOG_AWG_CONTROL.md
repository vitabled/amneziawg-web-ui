# AmneziaWG Control — Документация по обновлению

## Что изменилось

В панели управления серверами появился новый блок **AmneziaWG Control** на странице каждого сервера. Он позволяет управлять VPN-интерфейсом и смотреть live-трафик прямо из браузера — без необходимости заходить на сервер по SSH.

Помимо этого, логика деплоя нового сервера теперь автоматически устанавливает утилиту **amneziawg-tools** (`awg`, `awg-quick`) — официальный инструментарий AmneziaWG.

---

## Новый блок: AmneziaWG Control

Расположен на странице сервера (`/servers/{id}`) между секцией бэкапов и таблицей клиентов.

### Индикатор статуса

При открытии страницы панель автоматически проверяет реальное состояние интерфейса `wg0` внутри контейнера на удалённом сервере.

| Статус | Значение |
|--------|----------|
| 🟢 Running | Интерфейс поднят, VPN работает |
| 🔴 Stopped | Интерфейс опущен, VPN не работает |
| ⚪ Unknown | Не удалось получить статус (сервер недоступен) |

Кнопки Start/Stop блокируются автоматически в зависимости от текущего статуса — нельзя запустить уже запущенный интерфейс и наоборот.

### Кнопка Start VPN

Выполняет на удалённом сервере команду:
```
awg-quick up /opt/amnezia/awg/wg0.conf
```
Поднимает WireGuard-интерфейс `wg0` с параметрами обфускации AmneziaWG. После успешного запуска статус меняется на 🟢 Running.

### Кнопка Stop VPN

Выполняет:
```
awg-quick down /opt/amnezia/awg/wg0.conf
```
Корректно останавливает интерфейс. После остановки статус меняется на 🔴 Stopped.

### Кнопка Live Traffic

Запрашивает данные с сервера через команду:
```
awg show wg0
```
И отображает таблицу с информацией по каждому подключённому клиенту в реальном времени:

| Колонка | Описание |
|---------|----------|
| Peer (public key) | Первые 16 символов публичного ключа клиента. Зелёная точка — онлайн (хэндшейк < 5 минут назад) |
| Received | Трафик, полученный сервером от этого клиента |
| Sent | Трафик, отправленный сервером этому клиенту |
| Last Handshake | Время последнего успешного хэндшейка |
| Endpoint | IP-адрес и порт клиента |

> Данные не обновляются автоматически — нажми кнопку снова чтобы обновить.

## Обновлённый деплой сервера

При добавлении нового сервера через `/servers/create` и нажатии "Start Deployment" теперь автоматически выполняется установка **amneziawg-tools**.

### Что устанавливается

Репозиторий: [github.com/amnezia-vpn/amneziawg-tools](https://github.com/amnezia-vpn/amneziawg-tools)

Устанавливаются утилиты:
- `awg` — аналог `wg`, работает с AmneziaWG-конфигами
- `awg-quick` — аналог `wg-quick`, поднимает/опускает интерфейс с поддержкой обфускации

Установка происходит автоматически и поддерживает основные дистрибутивы Linux:
- **Debian / Ubuntu** — через готовый `.deb`-пакет или сборку из исходников
- **RHEL / CentOS / Fedora** — сборка из исходников
- **Другие дистрибутивы** — универсальная сборка из исходников

Если утилиты уже установлены на сервере — шаг пропускается.

### Почему это важно

До этого обновления панель использовала стандартные утилиты `wg` / `wg-quick`, которые не поддерживают параметры обфускации AmneziaWG (Jc, Jmin, Jmax, S1, S2, H1–H4). Теперь все операции — генерация ключей, запуск интерфейса, применение конфига, чтение статистики — выполняются через `awg` и `awg-quick`.

## Лимит трафика для клиентов

В таблицу `vpn_clients` добавлена колонка `traffic_limit`. Теперь при создании клиента можно установить лимит трафика — после его превышения клиент автоматически отключается.

Доступные варианты при создании клиента:
- Без лимита (по умолчанию)
- Готовые значения: 1, 5, 10, 25, 50, 100, 250, 500, 1000 GB
- Произвольное значение в МБ

В таблице клиентов отображается прогресс использования:
- Зелёный — использовано менее 80%
- Жёлтый — использовано 80–99%
- Красный **Over Limit** — лимит превышен, клиент отключён

## Технические детали

### Новые API-эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| `POST` | `/servers/{id}/start` | Запустить AmneziaWG |
| `POST` | `/servers/{id}/stop` | Остановить AmneziaWG |
| `GET` | `/servers/{id}/real-status` | Реальный статус интерфейса wg0 |
| `GET` | `/servers/{id}/traffic` | Live-трафик через awg show |
